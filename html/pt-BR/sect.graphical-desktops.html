13.3. Ambientes Gráficos

O campo de área de trabalho gráfica é dominado por duas coleções de software de grande porte: GNOME e KDE. Os dois são muito populares. Esse é mais um exemplo raro no mundo do software livre; o servidor web Apache, por exemplo, tem muito poucos similares.
Essa diversidade está enraizada na história. O KDE foi o primeiro projeto de ambiente de trabalho gráfico, mas ele escolheu usar o conjunto de ferramentas gráficas Qt e essa escolha não era aceitável por um grande número de desenvolvedores. O Qt não era software livre na época, e o GNOME foi iniciado com base no conjunto de ferramentas GTK+. O Qt se tornou software livre nesse meio tempo, mas os projetos não se fundiram e se desenvolveram em paralelo.
GNOME e KDE ainda trabalham juntos: sob a tutela da FreeDesktop.org, os projetos colaboraram em definir padrões para interoperabilidade entre as aplicações.
Escolher “o melhor” ambiente de trabalho gráfico é uma questão sensível, que nós preferimos apenas orientar. Nós iremos apenas descrever as muitas possibilidades e dar algumas opiniões para desencadear pensamentos. A melhor escolha irá ser a que você fizer após algumas experimentações.

13.3.1. GNOME

O Debian Wheezy inclui o GNOME versão 3.4, que pode ser instalado com um simples apt-get install gnome (ele também pode ser instalado selecionando a tarefa “Graphical desktop environment”).
O GNOME é notável por seus esforços em usabilidade e acessibilidade. Profissionais de design se envolveram na escrita de padrões e recomendações. Isso ajudou os desenvolvedores a criar interfaces gráficas de usuários satisfatórias. O projeto também recebe incentivo de grandes empresas de computação, como Intel, IBM, Oracle, Novell, e claro, várias distribuições Linux. Finalmente, muitas linguagens de programação podem ser usadas no desenvolvimento de aplicações com interface para o GNOME.
Levou algum tempo para o projeto GNOME construir essa infraestrutura, o que pode sugerir uma área de trabalho menos madura que o KDE. Os esforços de usabilidade e acessibilidade, em particular, são recentes, e os benefícios apenas começaram a aparecer nas últimas versões do ambiente.

Figura 13.1. O ambiente GNOME

Para administradores, GNOME parece ser melhor preparado para grandes implantações. A configuração da aplicação é administrada por dois registros, GSettings (o padrão atual, que guarda seus dados no DConf) e o GConf (o sistema antigo usado no GNOME 2.x, ainda utilizado por algumas aplicações GNOME 3.x). Esses registros podem ser consultados e editados com os comandos gsettings, dconf e gconftool-2, ou pelas interfaces gráficas dconf-editor e gconf-editor. Portanto, o administrador pode alterar a configuração dos usuários com um simples script. O site a seguir lista todas as informações de interesse para um administrador com a missão de gerenciar estações de trabalho GNOME:
→ http://library.gnome.org/admin/system-admin-guide/stable/
→ http://library.gnome.org/admin/deployment-guide/

13.3.2. KDE

O Debian Wheezy inclue a versão 4.8.4 do KDE, que pode ser instalado com apt-get install kde-standard.
O KDE teve uma rápida evolução baseado em uma abordagem muito "hands-on". Seus autores obtiveram resultados muito bons rapidamente, o que lhes permitiu crescer com uma grande base de usuários. Esses fatores contribuíram para a qualidade global do projeto. O KDE é um ambiente de área de trabalho perfeitamente maduro com uma ampla gama de aplicações.

Figura 13.2. O ambiente KDE

Desde o lançamento do Qt 4.0, o último problema de licença restante com o KDE não existe mais. Essa verssão foi lançada sob a GPL, tanto para Linux quanto para Windows (considerando que a versão Windows foi previamente lançada sob uma licença não-livre). Note que as aplicações KDE tem que ser desenvolvidas usando a linguagem C++.

13.3.3. Xfce e Outros

O Xfce é uma área de trabalho gráfica simples e peso leve, que é uma escolha perfeita para computadores com recursos limitados. Ele pode ser instalado com um apt-get install xfce4. Como o GNOME, o Xfce é baseado no conjunto de ferramentas GTK+, e vários componentes são comuns entre as duas áreas de trabalho.
Diferentemente do GNOME e KDE, o Xfce não almeja ser um projeto vasto. Além dos componentes básicos de uma moderna área de trabalho (gerenciador de arquivo, gerenciador de janelas, gerenciador de sessão, um painel para lançadores de aplicações e assim por diante), ele apenas provê algumas específicas aplicações: um navegador web muito peso leve (Midori), um terminal, um calendário, um visualizador de imagem, uma ferramenta para queimar CD/DVD, um reprodutor de mídia (Parole) e um controle de volume de som.

Figura 13.3. O ambiente Xfce

Outro ambiente de área de trabalho fornecido pelo Wheezy é o LXDE, que foca no aspecto "peso leve". Ele pode ser instalado com a ajuda do meta-pacote lxde.
