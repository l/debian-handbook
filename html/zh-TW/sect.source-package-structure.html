5.3. 原始套件的結構

5.3.1. 格式

原始檔案通常包括三個檔案，.dsc、.orig.tar.gz、與 .debian.tar.gz (或 .diff.gz)。它們允許從以程式語言撰寫的程式原始碼檔案，新增可執行套件 (如前所述的 .deb 檔案)。
.dsc (Debian Source Control) 檔案是包括 RFC 2822 標頭 (如同 control 檔案於 節 5.2.1, “描述：control 檔案”) 的簡短文字檔，描述原始套件及包括的其他檔案。由維護者簽署，保證其真實性。詳情見 節 6.5, “檢查套件真偽”。

需注意原始套件和可執行套件一樣有相依性 (Build-Depends)，指出編譯有問題軟體與建構其可執行套件的工具。

注意 不同的命名空間

需注意，原始套件與其產生的可執行套件間，不需使用對應的名稱。每個原始套件可能產生多個可執行套件。所以 .dsc 檔案有 Source 與 Binary 欄位記錄原始套件的名稱與其產生的可執行套件清單。

文化 為什麼分為多個套件

通常，一個原始套件 (供指定軟體使用) 可產生多個可執行檔。因為可以在不同的環境使用 (部份) 軟體。在共享程式庫裡，可以安裝於構成應用作用 (如，libc6)、或可以安裝以構成新的程式 (libc6-dev 然後形成正確的套件)。在主從服務裡發現同樣的邏輯，在一部機器安裝伺服器的內容同時在另部機器安裝客戶端的內容 (在此情況下，是 openssh-server 與 openssh-client)。
同樣的，文件可供專屬套件使用：使用者可以在軟體之外獨立安裝它，且隨時可以自磁碟空間中移除它。此外，也儲存在 Debian 鏡射的磁碟空間內，因為所有的架構都共享文件套件 (不必在各架構的套件中重複此文件)。

透視 原始套件的不同格式

本來祗有一種原始套件的格式。就是 1.0 格式，與 .orig.tar.gz 和 .diff.gz “debian 化” 套件有關 (還有一個變種，由一個 .tar.gz 檔案構成，若無 .orig.tar.gz 可用時自動取用它)。
因為 Debian Squeeze、及其發展者可以選擇新的格式以校正舊格式的問題。格式 3.0 (quilt) 在同個原始套件裡納入多個上游檔案：除了使用 .orig.tar.gz，還可納入 .orig-component.tar.gz 檔案。這種做法有助於散布在多個上游組件的軟體，祗需準備一個原始套件。通常把這些檔案壓縮成 bzip2 或 xz 格式而不是 gzip，以節省磁碟空間與網路資源。最後，在單件補丁的情況下，.diff.gz 被 .debian.tar.gz 取代，包括編譯指令與套件維護者奉獻的上傳套件補丁。最後的這些部份記錄於格式相容的 quilt — 它是強化補丁管理的工具。
.orig.tar.gz 檔案包括原發展者提供的原始碼。要求 Debian 的套件維護者不要修改它，以便檢查原來的檔案與整合後的檔案 (比較校驗碼就知道) 並且尊重其他著者的意願。
.debian.tar.gz 包括 Debian 維護者做的所有修訂，特別是新增至 debian 目錄執行 Debian 套件的指令。

工具 解壓縮原始套件

使用原始套件時，您可使用 dpkg-source 指令 (從 dpkg-dev 套件取得) 解壓縮它： 
$ dpkg-source -x package_0.7-1.dsc
可以使用 apt-get 下載原始套件並立即拆包它。需使用適當的 deb-src 位於 /etc/apt/sources.list 檔案內，然而 (詳情見 節 6.1, “寫入 sources.list 檔案”)。用於列出原始套件的 “來源” (就是收藏原始套件的伺服器)。 
$ apt-get source 套件

5.3.2. Debian 內的使用

Debian 的基礎是原始套件。Debian 所有的套件都從原始套件而來，對套件的修改也是針對原始套件。執行套件出了問題後，Debian 維護者修改原始套件後，再編譯為執行套件。維護者的工作成果都在 Debian 的原始套件內：您可輕易地回到原始套件找出源頭。
Debian 伺服器收到新版套件 (原始套件或執行套件) 時，原始套件是最重要的。原始套件被支援 Debian 的多種架構機器編譯。發展者同時送出指定架構 (通常是 i386 或 amd64) 的多個執行套件，其實一點都不重要，因為可以自動生成該等執行套件。
