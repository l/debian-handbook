2.4. 為什麼選擇 Debian 發行版？

選定 Linux 系列後，還要選發行版。同樣的，選擇很多。必須能夠使用多年，因為轉換系統極為費時費力又費錢 (雖然比在完全不同的系統間切換較為省事，如 Windows 或 OS X)。
可續性是必要的，必須保證有好幾年的定期更新與安全補丁。更新的時機很重要，有多部機器需要管理，Falcot 公司沒有辦法處理太頻繁的更新作業。因此，IT部門堅持執行最新的穩定版，以最好的技術支援，且保證取得安全補丁。實務上，通常祗保證指定年限的安全性更新。
最後，基於同質性與易於管理原因，必須在所有的伺服器 (部份為 Sparc 機器，目前執行 Solaris) 與辦公室電腦採用相同的發行版。

2.4.1. 商業與社群推動的發行版

Linux 發行版有兩個主要分類：商業驅動和社區驅動。前者由公司開發，與商業支援服務一同銷售。後者則是基於相同的開放開發模型開發而成的自由軟體。
因此，商業發行版傾向更頻繁地釋出發行版，給予更佳更新及相關服務的印象。它們的未來與公司的業績直接相關，不過，很多這種發行版已經消失 (Caldera Linux、StormLinux 等)。
社群發行版祗有自己的更新時間表。如同 Linux 核心，穩定後就發布新的發行版，永遠不會發布不穩定的版本。只要有足夠的個人發展者或第三方公司支援它，就能保證其存在。
從眾多發行版裡選取 Debian 的理由如下：
  • 它是社群發行版，不受任何商業行為限制；它的目標是技術導向，偏向控制產品的品質。
  • 在社群發行版中，各方面都是最優的：奉獻者、可用的套件數、以及持續存在的年限。社群的規模更是無可匹敵的。
  • 從過去的資訊分析，每隔 18 至 24 個月就有新的版本，都支援 5 年，管理者都贊同的時間表。
  • 針對法國自由軟體服務公司的調查發現，參與調查的公司都提供 Debian 的技術支援；同時，也是它們內部所使用的發行版。在服務提供者的多樣性是保持 Falcot 公司獨立不受特定廠商控制的重要原因。
  • 最後，Debian 支援眾多架構，包括 OpenPOWER 處理器的 ppc64el；因此，它可以裝載於 Falcot 公司最新的 IBM 伺服器上。

實務 Debian 長期支援

Debian 長期支援 (Long Term Support, LTS) 計畫始於2014年，旨在為所有的 Debian 穩定發行版提供 5 年的安全支援。由於 LTS 對大量部署 Debian 的組織極其重要，此計畫試圖從使用 Debian 的公司彙集資源。
→ https://wiki.debian.org/LTS
Falcot 公司不足以雇用一個全職的 IT 人員於長期支援計畫，因些，選擇與 Freexian 公司簽訂 Debian LTS 合約並支付服務費金。感謝此合約，Falcot 公司的管理者明白其使用的套件有問題時，將被優先處理且與長期支援團隊直接連繫。
→ https://wiki.debian.org/LTS/Funding
→ http://www.freexian.com/services/debian-lts.html
