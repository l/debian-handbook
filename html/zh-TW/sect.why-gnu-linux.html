2.3. 為什麼選擇 GNU/Linux 發行版？

基本Linux 或 GNU/Linux？

如您所知，Linux，祗是一個核心程式。所以，“Linux 發行版” 與 “Linux 系統”，都是不正確的稱呼：實際上是，以 Linux 為基礎發行版或系統。這種表示法未能提及搭配於核心之外的軟體，就是由 GNU 計畫發展出來的程式。理察·馬修·斯托曼博士創始該計畫，堅持使用 “GNU/Linux”，才能辨識出 GNU 計畫的重要貢獻及其自由的原則。
Debian 選擇遵從這些建議，並且以此命名其發行版 (所以，最新的穩定版是 Debian GNU/Linux 8)。
很多因素影響此抉擇。熟悉此發行版的系統管理員，確認 Debian 在全新電腦系統候選名單之內。儘管對公司的未來極為重要，困難的經濟情況與殘酷的競爭還是限制此作業的預算。這就是開放源碼方案勝出的原因：若干最近的研究指出，祗要有合格的人員執行，在相同或更佳服務品質的前提下，Debian 的花費小於專屬軟體。

實務整體擁有成本

整體擁有成本是花費在擁有或採購物件的總費用，在此案中指的是作業系統。此成本包括可能的授權費、使用新軟體的教育訓練、重置太慢的機器、額外的維修等。因為選定個案而直接發生的費用。
整體擁有成本係根據現況而來，抽離環境單獨來看就沒有意義。然而，在相同規則下，可以計算各選擇的整體擁有成本。此估價表是最重要的，很容易依此得出結論。所以，單個機器的整體擁有成本沒有意義，管理員的成本可以攤入其能管理的機器數，其數量與作業系統和工具有關。
眾多作業系統中，被IT部門列入候選名單的包括自由的 BSD 系統 (OpenBSD、FreeBSD、與 NetBSD)、GNU Hurd、與 Linux 發行版。GNU Hurd 還沒有釋出穩定版，立即被排除。祗剩下 BDS 與 Linux 可選。前者的優點很多，尤其在伺服器領域。然而，從實用的角度看，還是選擇 Linux 系統，因為它的安裝基礎與流行度都相當不錯而且也有很多正面的意義。其中包括很容易找到管理 Linux 機器的合格人員。尤有甚者，Linux 採用更新更快的硬體 (雖然差距不大)。最後，Linux 發行版多半伴隨親切的圖形使用者介面，初學者最需要它。

其他選擇 Debian GNU/kFreeBSD

Debian Squeeze 較特別，可以安裝在以 FreeBSD 為核心的 32 與 64 位元電腦；也就是 kfreebsd-i386 與 kfreebsd-amd64 架構。這些架構不是 “官方釋出的架構”，Debian 的套件祗有九成可用。
這種架構可能是 Falcot 公司管理者的恰當選擇，特別在防火牆 (核心程式支援三種不同的防火牆：IPF、IPFW、PF) 或 NAS (網路附加儲存系統，已測試及認可 ZFS 檔案系統) 方面。
