前書き

今や Linux は何年にもわたり力を蓄え続け、その高まる人気によってより多くのユーザを惹き付けています。Linux 人気の波に乗るための第一歩はディストリビューションを選ぶ事です。各ディストリビューションはそれぞれ異なる特徴を持っているため、ディストリビューションを選ぶことは重要です。最初から正しいものを選んでいれば将来の乗り換えコストを避ける事ができるのです。

BACK TO BASICS Linux ディストリビューション、Linux カーネル

厳密に言えば、Linux とはカーネル (ハードウェアとアプリケーションの間に位置するソフトウェアの中核部分) だけを指しています。
「Linux ディストリビューション」は完全なオペレーティングシステムです; これには通常、Linux カーネル、インストーラプログラム、そして最も重要な事ですがコンピュータを実際に役立つ道具に変えるためのアプリケーションとその他のソフトウェアが含まれています。
Debian GNU/Linux は多くのユーザにぴったりの「汎用」Linux ディストリビューションです。この本の目的は、ディストリビューションを選ぶ際に十分な情報に基づいた判断を下す事ができるように、Debian GNU/Linux の様々な側面を示す事です。

1. この本の目的とは?

CULTURE 商用ディストリビューション

多くの Linux ディストリビューションはある種の商業計画の下にディストリビューションを開発、販売する営利目的の企業から支援を受けています。例として、Canonical Ltd. が主に開発している Ubuntu; フランスの企業 Mandriva SA が開発している Mandriva Linux; Novell が保守と市販を行っている Suse Linux などが挙げられます。
その対極に、Debian や Apache ソフトウェア財団 (Apache ウェブサーバの開発を主導しています) のようなものがあります。Debian はフリーソフトウェア世界におけるすべてのプロジェクトを超越し、インターネットを介して共同作業するボランティア達によって実装されました。一部のボランティアはさまざまな企業から報酬を受ける仕事の一部として Debian に関する仕事を行っていますが、Debian プロジェクト全体は特定の企業との結びつきはありません。どんな企業でも完全にボランティアの貢献者が持っているものよりも強くプロジェクトの仕事について口を出す権利を持っていないのです。
ここ数年の Linux に関するマスコミ報道の数はかなり多いものです; このことは実在するマーケティング部門によって支援されているディストリビューション、言い換えれば企業支援を受けたディストリビューション (Ubuntu、Red Hat、SUSE、Mandriva など) の利益になっています。しかし Debian は決して重要度の低いディストリビューションではありません; ここ数年の複数の調査によれば、Debian はサーバとデスクトップの両方で広く使われている事が明らかになっています。これは Debian が主要な Linux ディストリビューションであるウェブサーバにおいて特に当てはまっています。

この本の目的は、このディストリビューションの良さを理解する一助になる事です。我々は 1998 年 (Raphaël) と 2000 年 (Roland) から開発者としてこのプロジェクトに参加して以降に集め続けた経験を共有したいと望んでいます。運が良ければ、あなたに我々の熱意が通じて、あなたがいつか我々の仲間になりますように…
この本の初版 (2004 年) は大きな穴を埋める目的がありました: Debian だけに注目したの最初のフランス語によって書かれた本でした。当時、この話題に関して書かれた他の多くの本はフランス語と英語の読者向けに書かれていました。残念なことに、それらのほとんどすべてが更新されておらず、何年も Debian に関する良い本がほとんどない状況が続いていました。我々は英語に翻訳されて (英語からさらに他の言語に翻訳されて) から新たな人生が始まったこの本がこの溝を埋めて多くのユーザの助けになる事を望んでいます。
