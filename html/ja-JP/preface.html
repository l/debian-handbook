序文

Debian は大きな成功を収めたオペレーティングシステムで、人々が想像したり気付いているよりも広くデジタルライフの至る所に浸透しています。これを明らかにするためにいくつかの例を挙げてみましょう。本書の執筆時点では、Debian は web サーバ用途で最も人気のある GNU/Linux 派生物です: W3Techs によれば、10% を超える web サーバが Debian で運用されています。考えてみてください: 今もし Debian が無かったら、いくつの web サイトが無くなってしまうのでしょうか? より興味深い展開として、Debian は 国際宇宙ステーションで使われるオペレーティングシステムに選ばれています。ソーシャルネットワークにある NASA や他の国際的組織のアカウントなどを通して、国際宇宙ステーションの宇宙飛行士の仕事に注目した事がありますか? Debian を使うことで彼らは業務を遂行したり、業務に関して投稿したりする事が可能になっています。数え切れないほど多くの企業、大学、行政機関が、業務を遂行したり世界中... さらに軌道上にいる何百万のユーザに向けてサービスを供給するために、日常的に Debian を利用しています!
Debian を使えばシステムを複雑にする事も多機能にする事も高い信頼性を持たせる事も可能ですが、Debian を選ぶことは単なるオペレーティングシステムを選ぶこと以上の意味を持っています。Debian は、日常生活においてソフトウェアに対する依存度がどんどん高まっているこの世界で人々はソフトウェアを自由に選択できるようになるべきである、という構想を持っています。Debian は、人々は自分のコンピュータを己の支配下におくべきでありコンピュータに支配されるべきではない、という基本的なフリーソフトウェア構想から生まれています。ソフトウェアに関して十分な知識を持っている人々は、自分にとって重要なすべてのソフトウェアを分解、改変、再構築してその成果を他の人と共有するべきです。ここで重要なのは、ソフトウェアの用途に関わらず成果を共有するという思想です。子猫の写真を Web に投稿するような取るに足らない作業に使われるソフトウェアも、車の制御や治療に使われる医療機器への電力供給などの潜在的に命にかかわるような目的に使われるソフトウェアも、そのソフトウェアに対して成された成果は同じように共有されます。さらに、ソフトウェアに関して深い知識を持っていない人々も同様にソフトウェアを自由に選択するべきです: そのような人々は自分たちの代わりに、ソフトウェアによって制御される機器の監査や改良を、自分たちが選んだ信頼できる人に委託するべきです。
人々がコンピュータを己の支配下におく上で、オペレーティングシステムがフリーであることは根本的な役割を担っています: オペレーティングシステムを支配しなければ、コンピュータデバイスを完全に支配できません。最高で完全にフリーなオペレーティングシステムを作成するという Debian の主要な目標はここから来ています。20 年以上もの間、Debian はフリーオペレーティングシステムを開発し、付随するフリーソフトウェア構想を推進し続けています。その際に Debian は世界中のソフトウェアフリーダム支持者に対して非常に高いハードルを設定し続けています。例えば、国際的標準機構や政府やその他のフリーソフトウェアプロジェクトが対象を「十分にフリー」と考えるべきか否かを決定する際には、通常ソフトウェア使用許諾に関する Debian の決定を基準にします。
しかしこの政治的な観点だけでは Debian の独自性をまだ十分に説明できません。Debian はその独立性に強く付随する非常に独特な社会実験でもあります。他の主要なフリーソフトウェアディストリビューションや人気のあるプロプライエタリオペレーティングシステムについてちょっと考えてみましょう。おそらくそれらは、プロジェクトの裏で開発を主に推し進めているかまたは控え目に言っても開発に無関係のすべての活動を請け負っている大企業と結びつける事ができるでしょう。Debian は違います。Debian プロジェクトでは、ボランティアが自分自身に Debian が維持され素晴らしいものであり続けるために必要なすべての活動に対する責任を課しています。ボランティア活動の範囲は驚くほど広いものです: その活動範囲は、翻訳からシステム管理、マーケティングから経営、カンファレンス組織からアートワークデザイン、簿記から法的問題など... に及び、ソフトウェアのパッケージ化と開発は言うまでもありません! Debian 開発者はこれらすべての面倒を見ているのです。
独立性を徹底的に守ることでもたらされる 1 番目の結果として、Debian は非常に多様なボランティアコミュニティを必要としておりそれに依存している、という点があります。上に挙げた技能およびその他の考えうる技能を使えば Debian に貢献したり、プロジェクトを改善する事ができるでしょう。Debian の独立性がもたらす 2 番目の結果として、Debian の方向性は特定企業の商業的興味によって決定されたものではない事が保障されている、という点があります。テクニカルニュースによって検証された数多くの最近の例の通り、Debian の興味は何も保証されておらず、その興味は常に人々がマシンを支配するという目標に向けられています。
Debian の独自性に対する貢献の最後の側面: 社会的実験の方法。お役所的と言われているにもかかわらず、実際のところは Debian の意思決定は全く体系化されておらず、ほとんど無秩序といえます。プロジェクトにおける責任の範囲は明確に定義されています。責任者は自身の責任が及ぶ範囲で自由に仕事を行う事が可能です。責任者はコミュニティが取り決める品質保証を守っている限り、責任者は仕事の内容や仕事のやり方に文句を言われることはありません。もしあなたが Debian のやり方に口を出したいのなら、あなた自身が主導して仕事を引き受ける心構えがなければいけません。このような能力主義の特異な形、時折これは do-ocracy と呼ばれます、は貢献者に強い力を与えます。十分な能力、時間、意欲を持っている人は誰でも、プロジェクトの方向性に大きな影響を及ぼす事ができるのです。Debian プロジェクトに参加している千人あまりの公式メンバーと世界中の数千の貢献者がこの点について証言しています。Debian が存在する内で最大のコミュニティ主導フリーソフトウェアプロジェクトと認められているのも少しも不思議なことではないのです。
そんなわけで Debian はとても独特です。これに気が付いているのは我々だけでしょうか? そんな訳がありません。DistroWatch によれば、世界には約 300 の活発なフリーソフトウェアディストリビューションがあります。そのうちの半分 (約 140) は Debian 派生物です。Debian 派生物とは、それらのディストリビューションが Debian を基にして、ユーザの必要性に合わせた改変 (通常はパッケージの追加、改変、再構築) を行い、その成果物をリリースしている事を意味しています。要するに派生物は、一つ一つのソフトウェアだけでなくディストリビューション全体としてコピーを改変したり再配布する自由が保障されているフリーソフトウェアを利用しているのです。派生ディストリビューションを通じて、新たなフリーソフトウェアのユーザと貢献者に働きかける潜在的可能性は大きなものです。今日、フリーソフトウェアは歴史的に見て乗り越える事が難しいと考えられている、例えば大きなデスクトップ環境などの、分野においてもプロプライエタリソフトウェアとようやく張り合う事ができるようになっています。これは主にに繁栄しているエコシステムのお陰だと考えられています。Debian は存在するフリーソフトウェアディストリビューションでは最大のエコシステムの根幹に位置しています: 直接 Debian を使っていなかったとしても、ディストリビュータが言わなかったとしても、今現在あなたは Debian コミュニティによる業績の恩恵を受けている可能性があります。
しかし Debian の独自性は時に思いもよらない結果を招く事があります。デジタルフリーダムに関する Debian の構想の結果、ソフトウェアの意味を再定義する必要性が出てきました。Debian プロジェクトはずっと以前から、オペレーティングシステムの一部として数多くのソフトウェアではない作品 (音楽、画像、文書、生データ、ファームウェアなど) を配布する必要がある、ということに気が付いていました。しかし、これらの作品に対してどのようにソフトウェアフリーダムを適用すればよいのでしょうか。個々の作品が独自の要件を設定するべきでしょうか、それともすべての作品に単一で高い水準の自由を求めるべきなのでしょうか? Debian プロジェクトは後者を選ぶことに決めました: Debian の一部として配布されるすべての作品はユーザに対して一義的な自由を提供するべきである。このような徹底的な哲学的姿勢は広範囲に影響を与えました。この姿勢は、フリーでないファームウェア、商業的環境で利用される事を許可していないアートワーク、(本の出版社が変わらない限り) 著者と出版社の評判を下げる事を回避するために改変できない本などは配布できないという事を意味しています。
あなたの手元にあるこの本は違います。この本は free as in freedom な本、つまりデジタルライフのあらゆる側面で Debian の自由利用基準に従う本です。とても長い間、Debian の自由利用基準に従う本が少ない点が Debian の重大な欠点でした。これは、Debian とその価値を広く伝え、同時に Debian の自由利用基準の価値を具体化してその利点を浮き彫りにするような読み物がほとんど存在しなかった事を意味しています。そして皮肉なことに、Debian それ自身の一部として配布する事ができる読み物がほとんど存在しなかった事も意味しています。この本は初めてこの欠点の克服に取り組んだ信頼できる本です。あなたはこの本を apt-get install したり、再配布する事が可能です。さらに将来他の人があなたの貢献の恩恵を受ける事ができるように、フォーク、改善、バグ報告、パッチ投稿を行う事が可能です。この本の「メンテナ」はこの本の著者でもあり、Debian プロジェクトのメンバーとして長期にわたって活動し続けています。我々著者は Debian のすべての側面に浸透しているフリーダムの精神に共感し、Debian の重要な部分に関する責任を負う事がどのような意味を持っているかを直接的に理解しています。このようなフリーな本を公開することは Debian コミュニティに対する素晴らしい貢献になります。
この Debian に関するフリーな読み物の基礎が皆さんにとって楽しいものである事を強く望んでいます。
2013 年 11 月
Stefano Zacchiroli (Debian プロジェクトリーダー 2010-2013) と Lucas Nussbaum (Debian プロジェクトリーダー 2013-現職)
