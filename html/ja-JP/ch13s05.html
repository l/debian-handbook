13.5. ウェブブラウザ

Epiphany は GNOME スイートのウェブブラウザで、Safari ブラウザ向けに Apple が開発した WebKit レンダリングエンジンを使います。対応するパッケージは epiphany-browser です。
Konqueror は KDE のファイルマネージャで、ウェブブラウザとしても機能します。Konqueror は KDE 特製の KHTML レンダリングエンジンを使います; KHTML は、Apple の WebKit が KHTML を下にしているという事実が証明しているように、素晴らしいエンジンです。Konqueror は konqueror パッケージに含まれます。
Epiphany や Konqueror では満足できないユーザは Iceweasel を使う事が可能です。Iceweasel は iceweasel に含まれ、Mozilla プロジェクトの Gecko レンダリングエンジンを使い、その上に簡素で拡張可能なインターフェースを備えています。

図13.7 Iceweasel ウェブブラウザ

CULTURE Iceweasel、Firefox、その他

多くのユーザは間違いなく、Debian Wheezy のメニューに Mozilla Firefox が無いことに驚きます。でも慌てる必要はありません: iceweasel パッケージは Iceweasel を含みます。Iceweasel は基本的に Firefox の別名と考えて差し支えありません。
名前変更の背後には、 Mozilla Foundation が定めた Firefox™ 登録商標の利用規則が満足できなかったという、論理的根拠があります: Firefox と名付けられたいかなるソフトウェアは公式の Firefox ロゴとアイコンを使わなければいけません。しかしながら、Firefox ロゴとアイコンはフリーライセンスでリリースされていないため、Debian は Firefox を main セクションで配布する事ができません。パッケージのメンテナは、ブラウザ全体を non-free に移動するのではなく、名前を変える事でこの問題を解決することにしました。
firefox コマンドは iceweasel パッケージから提供されますが、これは firefox コマンドを使おうとするツールとの互換性を考慮した措置です。
同様の理由で、Thunderbird™ 電子メールクライアントの名前も類似の Icedove に変更されました。

CULTURE Mozilla

Netscape Navigator はウェブの黎明期に標準的なブラウザでしたが、Microsoft Internet Explorer が登場すると徐々に後塵を拝するようになりました。この失敗に直面し、Netscape (企業名) は、第二の人生を歩ませるためにフリーライセンスのもとで Netscape を公開することにより、ソースコードを「フリー」にすることに決定しました。これが Mozilla プロジェクトの始まりです。長年の開発の後、結果は満足以上のものをもたらしました: Mozilla プロジェクトは (Gecko と呼ばれる) HTML レンダリングエンジンをもたらしました。これは最も厳密に規格に準拠するレンダリングエンジンです。Gecko レンダリングエンジンは特に Mozilla Firefox ブラウザで使われています。Mozilla Firefox はユーザ数を急速に伸ばしている、最も成功したブラウザです。
Wheezy には、ウェブブラウザ業界の新人である Chromium (chromium-browser パッケージから利用可能です) が含まれています。Chromium ブラウザは Google で急速に開発されました。その開発速度は Debian Wheezy の保守期間全体に渡って Chromium の単一のバージョンだけ保守する事が意味を成さなくなる程の速度です。Chromium の明確な目標は、ブラウザ性能を最適化しユーザの安全性を向上させることで、ウェブサービスをより魅力的にする事です。Chromium を構成するフリーコードは Google Chrome と呼ばれるプロプライエタリ版でも使われています。
